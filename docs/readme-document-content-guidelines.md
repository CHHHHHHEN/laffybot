# 文档撰写规则

本文档定义了 laffybot 项目所有技术文档的撰写规范，确保文档聚焦高层设计，排除实现细节。

---

## 1. 内容边界原则

### 1.1 核心原则

**文档应聚焦高层设计，排除实现细节。**

文档的目的是传达设计意图、架构决策和接口规范，而非替代代码。具体实现细节应在代码注释中说明。

### 1.2 应包含的内容

**设计层面**：
- 设计目标与动机
- 架构决策与理由
- 接口规范与契约
- 数据流与控制流
- 错误处理策略
- 状态转换规则

**抽象层面**：
- 概念模型
- 组件职责与边界
- 模块间交互方式
- 扩展点与扩展方式

### 1.3 不应包含的内容

**实现细节**：
- 具体代码实现或者代码示例
- 数据结构的具体定义
- 算法细节和优化技巧
- 内部函数的实现逻辑
- 变量命名和代码组织

**边际收益内容**：
- 性能监控方案
- 测试策略和测试用例
- 性能优化细节
- 资源管理细节（内存限制、连接池配置等）
- 可观测性方案（metrics、tracing）
- 未来拓展方向

**工程细节**：
- 部署配置
- 环境变量设置
- 构建流程
- CI/CD 配置
