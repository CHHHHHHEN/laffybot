# 文档撰写规则

本文档定义了 laffybot 项目所有技术文档的撰写规范，确保文档聚焦高层设计，排除实现细节。

---

## 1. 核心原则

**文档应聚焦高层设计，排除实现细节。**

文档的目的是传达设计意图、架构决策和接口规范，而非替代代码。具体实现细节应在代码注释中说明。

---

## 2. 应包含的内容

### 2.1 设计层面
- 设计目标与动机
- 预期实现目标
- 架构决策与理由
- 接口规范与契约
- 数据流与控制流
- 错误处理策略
- 状态转换规则

### 2.2 抽象层面
- 概念模型
- 组件职责与边界
- 模块间交互方式
- 扩展点与扩展方式

### 2.3 维护与演进
- 废弃功能/变量/代码的清理范围（如果存在）
- 日志分级设计/可观测性设计
- 参考文件（如果存在）

### 2.4 准确性与决策
- 标记不确定的点，验证后记录
- 待决策事项需明确标注"等待用户决策"

---

## 3. 不应包含的内容

### 3.1 实现细节
- 具体代码实现或代码示例
- 数据结构的具体定义
- 算法细节和优化技巧
- 内部函数的实现逻辑
- 变量命名和代码组织

### 3.2 低收益细节
- 性能监控方案
- 测试策略和测试用例
- 性能优化细节
- 资源管理细节（内存限制、连接池配置等）
- 未来扩展方向
- 收益小且维护成本高的 fallback 代码

### 3.3 工程配置
- 部署配置
- 环境变量设置
- 构建流程
- CI/CD 配置

---

## 4. 文档修订规则

- 修订已有文档时，严禁全部覆写，必须逐句修改，保留有价值的既有内容。
