# Laffybot UI 设计规范

> **文档范围说明**：本文档定义 Laffybot UI 的视觉规范、交互模式和设计原则，确保所有 UI 组件和页面在视觉和交互上保持一致。
>
> **适用范围**：本文档适用于所有 Laffybot UI 的设计和开发工作，包括 Web UI 和未来可能的桌面端 UI。具体组件职责和交互细节见 `ui-design.md`，技术选型见 `ui-tech-selection.md`。

## 设计原则

### 四个核心原则

1. **清晰胜过花哨** — 每个界面元素都有明确目的。不添加装饰性元素，避免过度设计。聊天界面尤其需要减少视觉干扰，让内容成为焦点。

2. **状态可见** — 用户永远知道当前正在发生什么：加载中、流式生成中、已完成、出错、空闲。每个状态都有明确的视觉反馈，不依赖用户猜测。

3. **渐进呈现** — 复杂信息（推理过程、工具调用参数、代码内容）默认折叠或摘要展示，用户可按需展开。不在一屏内展示所有细节。

4. **操作可逆** — 所有破坏性操作（删除会话等）都有确认步骤。非破坏性操作支持乐观更新，失败时可回滚恢复。

## 色彩系统

### 中性色板

中性色构成界面基础骨架。浅色模式下值如下，暗色模式映射见下文：

| 用途 | 浅色色值 | 描述 |
|------|----------|------|
| 页面背景 | 白 (#FFFFFF) | 主背景色 |
| 次级背景 | 浅灰 (#F5F5F5) | 侧边栏、消息气泡、卡片背景 |
| 悬停背景 | 中浅灰 (#EBEBEB) | 列表项、卡片悬停态 |
| 边框分割 | 中灰 (#E5E5E5) | 分割线、卡片边框、输入框边框 |
| 主文字 | 深灰 (#1A1A1A) | 正文、标题 |
| 次要文字 | 中灰 (#666666) | 辅助信息、时间戳、描述文字 |
| 占位文字 | 浅灰 (#999999) | 输入框 placeholder、空状态提示 |
| 禁用态 | 更浅灰 (#CCCCCC) | 禁用按钮、禁用输入框 |

品牌色用于突出用户归属和主要操作。

品牌色已选定为 Indigo（#6366F1），位于蓝紫色相范围，满足 WCAG AA 对比度要求。

| 用途 | 色值（浅色） | 色值（暗色） |
|------|-------------|-------------|

| 悬停/激活态 | #4F46E5 | #6366F1 |

## 阴影

| 层级 | 用途 | 效果描述 |
|------|------|----------|
| 无阴影 | 页面层元素 | — |
| 低层 | 默认卡片 | 单向轻度下投阴影（垂直偏移、半透明黑色），区别卡片与背景 |
| 中层 | 下拉菜单、浮动弹出 | 双向均匀扩散阴影（偏移和模糊半径大于低层），视觉上浮于内容之上 |
| 高层 | 模态对话框 | 最大范围均匀扩散阴影，配合半透明遮罩，视觉层级最高 |

## 交互模式

### 悬停反馈

所有可交互元素在悬停时应有视觉反馈：

| 元素类型 | 悬停效果 |
|----------|----------|
| 文字按钮/链接 | 颜色变化或在文字下方加下划线 |
| 填充按钮 | 背景色加深一个层级 |
| 卡片/列表项 | 背景色变为悬停色，或左侧出现高亮边框 |
| 图标按钮 | 背景出现圆形/方形浅色遮罩 |

### 焦点状态

所有可聚焦元素在键盘导航聚焦时显示清晰的外环轮廓（outline），颜色与品牌色一致。鼠标点击时不显示聚焦环。

### 过渡动画

| 场景 | 时长 | 缓动曲线 | 说明 |
|------|------|----------|------|
| 瞬时反馈（悬停、点击） | 150ms | ease-out | 颜色变化、背景变化 |
| 组件出现/消失（Toast） | 200ms | ease-out | 透明度 + 位移 |
| 面板展开/折叠（Sidebar） | 250ms | ease-in-out | 宽度过渡 |
| 浮动按钮出现 | 200ms | ease-out | 透明度 + 上移 |
| 对话框打开/关闭 | 200ms | ease-out | 透明度 + 缩放 |

所有动画应遵循 `prefers-reduced-motion` 媒体查询，用户开启简化动效时过渡时长归零，所有动效回退为瞬时状态切换。

### 通知反馈

通知（Toast）用于非阻塞地反馈操作结果：

| 属性 | 规格 |
|------|------|
| 布局位置 | 页面顶部居中，水平居中，位于内容层之上 |
| 最大同时显示数 | 1 条（新通知替换当前显示中的通知） |
| 自动消失时长 | 成功/信息类 3 秒，错误类需用户手动关闭 |
| 出现/消失动画 | 透明度过渡 + 向下位移，时长 200ms，ease-out |
| 类型区分 | 成功（绿色语义色图标）、错误（红色语义色图标）、信息（蓝色语义色图标） |
| 交互 | 不阻塞用户操作，可点击关闭 |

### 加载反馈

| 场景 | 反馈类型 | 说明 |
|------|----------|------|
| 会话列表初次加载 | 骨架屏 | 3-4 个灰色占位条，宽度在 60%-90% 之间随机，模拟文本行长度差异 |
| 消息历史加载 | 居中旋转指示器 | 位于消息区域中央 |
| 页面/区域刷新 | 内联指示器 | 不遮挡页面内容 |
| 流式 token 等待 | 闪烁光标 | 最后一条助手消息末尾的竖线光标 |

## 布局规范

### 响应式断点

| 断点 | 宽度范围 | 布局调整 |
|------|----------|----------|
| 桌面 (desktop) | >= 1024px | 标准双栏布局，Sidebar 常开 |
| 平板 (tablet) | 768px - 1023px | Sidebar 默认折叠，可展开为覆盖层 |
| 手机 (mobile) | < 768px | Sidebar 以抽屉形式从左侧滑入覆盖内容 |

### 内容宽度约束

- 消息列表最大展示宽度 800px，两侧自动留白
- 设置面板最大展示宽度 720px
- 对话框宽度：小号 400px，大号 560px

### z-index 层级

| 层级 | 用途 |
|------|------|
| 1 | Sidebar、页面内容 |
| 10 | Sidebar 折叠后的覆盖层（tablet/mobile） |
| 20 | 浮动按钮（回到最新） |
| 30 | 下拉菜单、弹出面板 |
| 40 | 对话框遮罩 + 对话框 |
| 50 | Toast 通知 |
| 60 | 全局加载指示器 |

## 滚动行为

### 消息列表滚动

| 场景 | 行为 |
|------|------|
| 新消息到达且用户位于底部 | 自动滚动到底部，平滑过渡 |
| 用户已回看历史消息（离开底部） | 停止自动滚动，底部显示"回到最新"浮动按钮 |
| 点击"回到最新" | 平滑滚动至最新消息 |
| 加载历史消息（向上滚动触发） | 滚动位置保持不变，内容向上增长 |

### 滚动条

- 使用系统默认滚动条样式，不自定义
- 内容区域始终可垂直滚动

## 文案规范

### 语气

友好、直接、不废话。避免过度正式或过于随意的表达。

### 通用规则

- **错误消息**：说明"出了什么问题" + "用户可以怎么做"，不展示技术细节（不显示错误码、状态码、栈信息）
- **空状态**：说明"这里还没有内容" + 引导操作（如"创建第一个会话"）
- **按钮标签**：使用动词（"新建会话"、"发送"、"取消"），避免使用名词作为操作动词
- **确认文案**：明确表述后果（"删除后将无法恢复"），不使用模糊表述（如"确定要删除吗？"）

### 示例

| 场景 | 文案 |
|------|------|
| 会话列表空状态 | 还没有会话，创建第一个会话开始对话 |
| 消息历史加载失败 | 消息加载失败，点击重试 |
| 删除确认 | 删除后将无法恢复该会话及其所有消息 |
| 取消发送确认 | 取消后将丢弃当前已生成的内容 |
| 会话 busy 提示 | 当前会话正在处理中，请等待完成 |
| 流式中断提示 | 连接已断开，部分内容可能未完成，点击重新发送 |

## 图标规范

- 使用 lucide-react 图标库，保持图标风格一致
- 图标尺寸：导航图标 20px，内联图标 16px，状态图标 14px
- 按钮中的图标与文字间距为 6px
- 图标本身不承载独立交互含义，需配合文字标签使用

## 表单设计模式

| 场景 | 验证方式 | 错误展示 |
|------|----------|----------|
| 输入框 | 提交时验证 | 输入框下方内联错误文字，红色 |
| 对话框表单 | 提交时验证 | 表单区域顶部全局错误 + 出错字段高亮 |
| 设置项 | 保存时验证 | 保存按钮旁错误提示 |

- 输入框在用户开始输入前不显示验证状态
- 验证错误不阻止用户继续编辑
- 保存操作应有成功/失败的明确反馈（Toast 或状态变化）

## 可访问性

### 对比度

- 所有文字和图标与背景的对比度需满足 WCAG AA 标准（普通文字 ≥4.5:1，大号文字 ≥3:1）
- 品牌色和语义色选定后需验证对比度合规，不合规时调整色值或仅用于非文字元素

### 触控交互（移动端）

- 所有可触控元素的最小触控区域为 44×44px（可包含内边距和外边距）
- 图标按钮的实际触控面积需大于视觉尺寸

### 辅助技术

- 图标按钮和纯图标操作需提供等效文字标签（供屏幕阅读器使用）
- 状态变化（加载、成功、失败）需通过非视觉方式传达，不依赖颜色作为唯一区分手段
- 推理过程、工具调用等折叠区域需标明展开/折叠状态

### 键盘导航

- 所有交互功能必须可通过键盘独立完成
- 对话框打开时聚焦到第一个可交互元素，关闭后焦点回到触发元素
- 对话框使用焦点陷阱（focus trap）

## 实现状态注释

### 已实现的规范项目
- ✅ 色彩系统：中性色板、品牌色（Indigo）、语义色均已通过 CSS 变量定义在 `index.css` 中
- ✅ 字体排版：系统无衬线 / 等宽字体栈，字号层级均已映射
- ✅ 间距系统：使用 4px 基数的间距层级
- ✅ 圆角：sm/md/lg/full 均通过 CSS 变量定义
- ✅ Z-index 层级：6 层（1/10/20/30/40/50/60）均已定义
- ✅ 滚动行为：消息列表自动跟随 + "回到最新"浮动按钮
- ✅ 骨架屏：会话列表首次加载显示 3 条灰色动画占位
- ✅ 流式光标：StreamMessage 末尾闪烁光标
- ✅ 过渡动画：悬停 150ms、Sidebar 250ms、Toast 200ms 等
- ✅ 简化动效：`prefers-reduced-motion` 媒体查询已适配
- ✅ 图标：使用 lucide-react，尺寸遵循规范
- ✅ 对话框焦点：ConfirmDialog / NewSessionDialog 自动聚焦 + Escape 关闭

### 未实现或部分实现的规范项目
- ⚠️ 暗色模式：CSS 变量已定义（`.dark` class），但 `ui-store.theme` 状态未接入实际的 class 切换逻辑，当前仅通过系统 `prefers-color-scheme` 自动适配（若用户手动配置了 class 切换机制则可工作）
- ⚠️ 阴影层级：仅"回到最新"按钮使用了 `shadow-lg`，未定义精细的阴影变量
- ⚠️ 响应式断点：Sidebar 在 lg 以下表现为覆盖层，但 tablet（768-1023px）和 mobile（<768px）的区分行为未严格实现
- ⚠️ 触控区域 ≥ 44×44px：未系统验证
- ⚠️ aria-live：流式渲染区域未标记 `aria-live="polite"`
- ❌ 品牌色悬停态品牌色用户消息气泡背景色（见 `ui-design.md` 组件颜色分配表）已在 CSS 中定义为 `--color-brand-light`
- ❌ 设置面板配色仅在 CSS 变量中定义，面板本身尚无可工作的管理 API 支持
